1️⃣ Đánh giá hiện trạng (đến thời điểm 18:06 30 / 06 / 2026)
Thành phần	Trạng thái	Nhận xét chi tiết
Frontend (HTML + CSS + JS)	Cơ bản – còn nhiều hạn chế	- Giao diện sử dụng Bootstrap‑like nhưng chưa áp dụng các nguyên tắc thiết kế hiện đại (glassmorphism, gradient, micro‑animation).
- Dark mode: màu chữ trong chế độ tối còn mờ, khó đọc.
- Modal hiển thị chi tiết phòng trên mobile có height:100vh → nội dung bị cắt, không thể scroll.
- Các thành phần filter, dropdown, toggle chưa có hiệu ứng mượt, chưa tối ưu cho cảm ứng.
Backend (PHP)	Ổn định, có một số cải tiến	- Đã thêm polyfill cho mbstring tránh lỗi môi trường không có extension.
- Thêm hàm getApproximateCoords để tự động tính tọa độ dựa trên địa chỉ, tránh các ghim trùng & điều chỉnh vị trí trung tâm trường Đại học Kinh tế Nghệ An.
- Cột lat, lng trong DB được tạo tự động nếu chưa tồn tại.
- Tuy nhiên không có lớp trừu tượng cho việc truy vấn, chưa có chuẩn MVC hoặc ORM.
Cơ sở dữ liệu	Đủ cấu trúc cho MVP	- Bảng phongtro và dangbai_chothuetro có các trường cần thiết (id, tiêu đề, mô tả, hình ảnh, giá, diện tích, địa chỉ, lat/lng, trạng thái).
- Thiếu index trên cột diachi, lat, lng gây performance khi lọc theo khu vực.
Bản đồ (Leaflet + MarkerCluster)	Hoạt động nhưng còn lẻ	- Các pin hiện tại dựa trên lat/lng từ DB hoặc hàm dự đoán, đã khắc phục việc pin trùng bằng offset ngẫu nhiên.
- Chưa có tích hợp bản đồ vệ tinh (đã có toggle, nhưng thumbnail chưa đồng bộ với vị trí mới).
- Không có clustering hiệu quả khi số lượng phòng > 200 (độ trễ khi zoom).
Chatbot & AI (Gemini / Groq)	Chưa triển khai	- Đề xuất: API riêng cho Gemini (xác thực ảnh) và Groq (trả lời ngôn ngữ).
- Hiện tại chưa có endpoint trong api/ để giao tiếp với AI, chưa có key management qua .env.
Accessibility & SEO	Cơ bản	- Thiếu <title>, <meta description>, thẻ heading hợp lý.
- Các hình ảnh chưa có alt mô tả đầy đủ.
- Không có schema.org cho địa điểm.
Performance	Chưa tối ưu	- Tải toàn bộ JS/CSS trong một file, không có code splitting.
- Không dùng caching (browser cache, HTTP cache).
- Ảnh chưa được nén, dùng URL placeholder có kích thước lớn.
Bảo mật	Cần cải thiện	- Các khóa API phải được lấy từ .env – đã có đề xuất nhưng chưa thực hiện.
- Không có CSRF token cho các request POST (đăng tin, đặt phòng).
- Truy cập DB bằng PDO nhưng chưa prepared statements cho mọi query (có một số).
Kiểm thử	Thiếu	- Không có test unit hoặc integration.
- Kiểm thử UI chỉ bằng manual trên desktop.

3️⃣ Đề xuất cải tiến hiện đại – chuẩn bị cho cuộc thi cấp Quốc gia
3.1. Kiến trúc & Backend
Đề xuất	Lý do	Công cụ / Công nghệ
Micro‑service cho AI: tách riêng API gemini (xác thực ảnh) và groq (trả lời chatbot) thành các service Docker.	Dễ mở rộng, độc lập, có thể scale riêng.	Docker, Symfony/Lumen (PHP micro‑framework) hoặc Node.js (Express)
REST + GraphQL cho dữ liệu phòng: cung cấp query linh hoạt (filter, pagination).	Độ linh hoạt cho frontend, giảm tải.	Lighthouse, GraphQL‑PHP hoặc Apollo Server
ORM (Eloquent hoặc Doctrine)	Quản lý DB an toàn, giảm SQL injection, tự động migrations.	Laravel Eloquent (có thể dùng Lumen)
Caching: Redis cache kết quả filter, API trả lời AI.	Tăng tốc độ phản hồi, giảm chi phí API.	Redis, PHP Predis
CI/CD: GitHub Actions + Docker Compose để tự động build, test, deploy.	Đảm bảo chất lượng, hỗ trợ quy trình thi.	GitHub Actions, Docker Hub
3.2. Frontend & UI/UX
Đề xuất	Lý do	Công nghệ
Framework hiện đại: React + Vite (hoặc Vue + Vite).	SPA mượt, hỗ trợ component reuse, hot reload.	Vite, React 18, TailwindCSS (nếu muốn, nhưng cần hỏi người dùng).
Thiết kế Premium: glassmorphism, gradient, micro‑animation (Framer Motion), dark‑mode tùy chỉnh.	Tạo ấn tượng “đẳng cấp”, đáp ứng yêu cầu thi.	CSS3 (backdrop-filter), @keyframes, Framer Motion (React)
Responsive Modal: thay height:100vh bằng max-height:85vh + overflow-y:auto; dùng CSS variable để hỗ trợ dark mode.	Giải quyết vấn đề cắt nội dung trên mobile.	CSS custom properties
Lazy‑load ảnh & Video + WebP + Image CDN (Cloudinary).	Giảm thời gian tải, tối ưu performance.	loading="lazy" attribute, Cloudinary API
PWA: manifest, service worker, offline cache.	Đánh dấu “ứng dụng hiện đại”, cho phép sử dụng khi không có mạng.	Workbox, Vite‑PWA plugin
Accessibility: ARIA roles, keyboard navigation, contrast check.	Điểm cộng lớn trong tiêu chí thi.	axe‑core, ESLint a11y plugin
3.3. SEO & Marketing
Đề xuất	Lý do
Thêm meta tags (title, description, og:image) cho mỗi page.	
Schema.org cho Place (địa chỉ phòng trọ) và Product (giá thuê).	
Sử dụng sitemap.xml và robots.txt.	
Tối ưu core web vitals (LCP, FID, CLS) bằng tối ưu tài nguyên tĩnh.	
3.4. Bảo mật
Đề xuất	Lý do
Environment variables: mọi API key (Gemini, Groq, DB) qua .env.	
CSRF token cho mọi form POST.	
Sanitize & escape đầu ra HTML (đã có htmlspecialchars, nhưng cần audit toàn bộ).	
Rate limiting cho API AI để tránh bị chặn.	
HTTPS (self‑signed cert trong dev, Let's Encrypt cho prod).	
Content Security Policy (CSP) để ngăn XSS.	
3.5. Kiểm thử & Đánh giá
Đề xuất	Công cụ
Unit tests cho PHP (PHPUnit) và JS (Jest).	PHPUnit, Jest
End‑to‑end tests với Cypress (đặc biệt cho map, modal, filter).	Cypress
Performance testing bằng Lighthouse (CI).	Lighthouse CI
Accessibility testing bằng axe‑core (CI).	axe‑core
