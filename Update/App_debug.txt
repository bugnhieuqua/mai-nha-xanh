================================================================================
BẢN HƯỚNG DẪN CẤU HÌNH VÀ SỬ DỤNG BIẾN APP_DEBUG TRÊN MÁI NHÀ XANH
================================================================================

Tài liệu này hướng dẫn chi tiết cách thức hoạt động, các thay đổi đã thực hiện và cách chạy thử nghiệm thực tế đối với biến cấu hình APP_DEBUG trong tệp .env.

--------------------------------------------------------------------------------
I. SƠ ĐỒ HOẠT ĐỘNG CỦA APP_DEBUG
--------------------------------------------------------------------------------

Khi người dùng truy cập bất kỳ trang nào (ví dụ: index.php):

               [ Người dùng truy cập index.php ]
                              │
                              ▼
                      [ Gọi bootstrap.php ]
                              │
                              ▼
                    [ Gọi error_handler.php ]
                              │
                              ▼
             [ Đọc cấu hình APP_DEBUG từ file .env ]
                              │
               ┌──────────────┴──────────────┐
               ▼                             ▼
       [ APP_DEBUG = true ]          [ APP_DEBUG = false ]
        (Chế độ phát triển)          (Chế độ chạy trên Host)
               │                             │
               ▼                             ▼
       [ Bật hiển thị lỗi ]          [ Ẩn hoàn toàn lỗi ]
     (display_errors = 1)          (display_errors = 0)
               │                             │
               ▼                             ▼
       [ Khi lỗi xảy ra ]            [ Khi lỗi xảy ra ]
               │                             │
               ▼                             ├────────────────────────┐
       [ In lỗi trực tiếp ]                  ▼                        ▼
       ra trình duyệt (dòng,        [ Hiện trang lỗi ]        [ Ghi âm thầm vào ]
       file, stack trace...)        bảo mật thân thiện         tệp php_errors.log
                                     "HỆ THỐNG BẢO TRÌ"

--------------------------------------------------------------------------------
II. NHỮNG GÌ ĐÃ LÀM (ỨNG DỤNG CỦA APP_DEBUG TRONG HỆ THỐNG)
--------------------------------------------------------------------------------

Biến APP_DEBUG được quản lý tập trung và áp dụng cho các phần sau trong dự án:

1. Thiết lập hiển thị/ẩn lỗi PHP:
   - File áp dụng: includes/error_handler.php
   - Khi bật (true): Bật display_errors và hiển thị giao diện báo lỗi chi tiết dạng Hộp đỏ (Error Box) kèm Stack Trace.
   - Khi tắt (false): Tắt display_errors, hiển thị trang lỗi thân thiện và ghi log thầm lặng vào logs/php_errors.log.

2. Tự động tắt xác thực SSL cho các cuộc gọi AI (Gemini / Groq) trên localhost:
   - Các file áp dụng: 
     * api/v2/chat.php (Chatbot AI)
     * api/v2/rooms_search.php (Tìm kiếm thông minh)
     * includes/ai_moderation_helper.php (Kiểm duyệt bài đăng)
     * api/ai_autofill.php (Điền nhanh thông tin)
   - Tác dụng: Khi chạy trên localhost (APP_DEBUG=true), cURL sẽ tự động tắt kiểm tra chứng chỉ SSL (CURLOPT_SSL_VERIFYPEER = false) để tránh lỗi kết nối SSL của PHP trên hệ điều hành Windows. Khi đưa lên Host chạy thật (APP_DEBUG=false), cURL sẽ yêu cầu xác thực SSL nghiêm ngặt nhằm bảo mật dữ liệu tối đa.

--------------------------------------------------------------------------------
III. CÁCH DÙNG VÀ KIỂM THỬ THỰC TẾ
--------------------------------------------------------------------------------

1. Cách chuyển đổi trạng thái:
   - Mở file .env ở thư mục gốc.
   - Thay đổi giá trị tại dòng cấu hình APP_DEBUG:
     * Chạy thử nghiệm trên máy cá nhân: APP_DEBUG=true
     * Đưa lên chạy thực tế trên internet: APP_DEBUG=false

2. Cách chạy test lỗi thực tế trên trang chủ (index.php):
   Chúng tôi đã tích hợp một cổng kiểm thử an toàn ngay trên trang chủ index.php. Hãy truy cập đường dẫn sau trên trình duyệt:
   --> http://localhost/mai-nha-xanh/index.php?trigger_test_error=true

   - Trường hợp APP_DEBUG=true:
     * Trình duyệt của bạn sẽ hiển thị chi tiết dòng lỗi Fatal Error của PHP:
       "Fatal error: Uncaught Error: Call to undefined function goi_ham_khong_ton_tai() in index.php on line 13"
     * Mục đích: Giúp bạn dễ tìm ra tệp tin và số dòng bị lỗi để sửa ngay lập tức.

   - Trường hợp APP_DEBUG=false:
     * Trình duyệt ẩn lỗi và chỉ hiển thị thông báo:
       "🛠️ HỆ THỐNG ĐANG BẢO TRÌ. Vui lòng quay lại sau ít phút."
     * Hãy mở tệp tin logs/php_errors.log để kiểm tra. Bạn sẽ thấy dòng lỗi thật sự đã được lưu trữ an toàn tại đây:
       "FATAL ERROR: Call to undefined function goi_ham_khong_ton_tai() in index.php:13"
     * Mục đích: Giữ bí mật mã nguồn dự án trước hacker.

3. Cách chạy test cURL SSL đối với Chatbot:
   Truy cập tệp hỗ trợ: http://localhost/mai-nha-xanh/test_debug.php
   - Nếu APP_DEBUG=true: Kết nối cURL tới Google API và Groq API báo màu xanh "Thành công".
   - Nếu APP_DEBUG=false: Kết nối cURL báo lỗi đỏ "cURL Error: SSL certificate problem" (Do localhost thiếu chứng chỉ CA). Điều này chứng minh cơ chế bảo vệ SSL đã được bật nghiêm ngặt.
